# 트랜잭션 (Transaction)

## 개념

- 트랜잭션은 특정 기능이 아니라, 여러 데이터 연산을 하나의 논리적 작업 단위로 묶어서 처리하기 위한 데이터베이스의 핵심 개념임
- MySQL, Oracle, SQLite 등 대부분의 DBMS가 트랜잭션 기능을 제공하며, 세부 동작과 구현 방식은 DBMS마다 조금씩 다름

## 목적

- 지키려는 것: 데이터의 일관성 / 무결성 (어느 시점에 조회해도 정해진 규칙이 깨지지 않은 상태)
- 지키는 방법: 여러 작업을 전부 아니면 전무(all-or-nothing)로 처리함
- 추가 보장: 커밋된 결과는 장애가 발생해도 사라지지 않음 (지속성)

## ACID — 트랜잭션이 보장하는 4가지 성질

트랜잭션을 설명할 때 거의 항상 함께 나오는 개념임.

- **Atomicity(원자성)**: 전부 실행되거나 전부 실행되지 않음. 부분 반영이 없음
- **Consistency(일관성)**: 트랜잭션 전후로 데이터가 정해진 규칙(제약조건 등)을 항상 만족함
- **Isolation(격리성)**: 여러 트랜잭션이 동시에 실행돼도 서로 간섭하지 않음. 각자 단독으로 실행되는 것처럼 보임
- **Durability(지속성)**: 커밋된 결과는 시스템이 중단돼도 사라지지 않음 (디스크 / 로그에 기록됨)

⇒ 사실상 Consistency(일관성)를 지키기 위해 나머지 A·I·D를 준수하는 구조로 볼 수 있음

## COMMIT과 ROLLBACK

- **COMMIT**: 트랜잭션 안의 모든 작업을 최종 확정함. 이 시점부터 결과가 영구 반영됨
- **ROLLBACK**: 문제가 생기면 트랜잭션 시작 전 상태로 전부 되돌림

⇒ 이 외에도 TCL(Transaction Control Language)에 트랜잭션을 제어하는 명령들이 포함됨

## 정리

- 핵심 골격: `BEGIN → (작업들) → COMMIT / ROLLBACK`
- SAVEPOINT 계열은 "부분 되돌리기"가 필요할 때 사용하는 보조 도구임
