# Redis 정리 (2026-07-22)

## 1. Redis 핵심 개념

- Redis는 **인메모리(in-memory) key-value 저장소**임
- 데이터를 디스크가 아닌 **RAM(메모리)** 에 올려두고 다루는 것이 핵심 특징임
- 저장 구조는 `key → value` 형태의 단순한 쌍임 (예: `user:1001 → "현주"`)
- 메모리에서 처리하므로 읽기/쓰기 응답이 매우 빠름 (마이크로초 단위)
- SQL 같은 복잡한 조인·질의는 없고, key로 값을 바로 꺼내는 방식임

## 2. 다른 저장소와의 비교

| 구분 | Redis | MongoDB | MySQL(관계형) |
|---|---|---|---|
| 저장 위치 | 주로 메모리 | 주로 디스크 | 디스크 |
| 데이터 구조 | 단순 key-value | 문서(JSON 형태) | 테이블(행/열) |
| 주 용도 | 빠른 캐시·임시 저장 | 데이터 영구 저장 | 데이터 영구 저장 |
| 질의 방식 | key로 바로 조회 | 필드 조건 검색 | SQL 조회·조인 |

- Redis는 "속도가 중요한 곳에 쓰는 빠른 보조 저장소" 성격임
- MongoDB는 MySQL 자리를 대신할 수 있는 메인 DB 성격임 (Redis와는 역할이 다름)

## 3. 리소스 부담 (Airflow와의 차이)

- Redis 실행 자체는 가벼움 → 단일 프로세스로 도는 단순 저장소이기 때문임
- 다만 데이터를 메모리에 올리므로, **담는 데이터 양만큼 RAM을 그대로 차지함**
  - 10GB 데이터를 넣으면 RAM 10GB를 사용하는 구조임
- Airflow는 스케줄러·웹서버·워커·메타DB·메시지 브로커 등 여러 프로세스가 상시 떠 있는 복합 시스템이라 무거움
  - 이때 메시지 브로커로 Redis가 자주 쓰임 → Airflow 안에서 Redis는 오히려 가벼운 부품 하나임
- 메모리 부담 때문에 실무에서는 자주 쓰고 빠른 응답이 중요한 데이터만 골라 담고, TTL(만료시간)로 관리함

## 4. persistence (재시작 시 초기화 여부)

- Redis에는 디스크 백업(persistence) 기능이 있음 → "무조건 껐다 켜면 초기화"는 정확하지 않음
- 두 가지 방식이 존재함
  - **RDB**: 주기적으로 메모리 전체를 스냅샷 떠서 `dump.rdb` 파일로 저장함
  - **AOF**: 실행된 쓰기 명령을 로그 파일에 계속 기록함
- persistence를 켜면 데이터 파일이 실제로 생성되고, 재시작해도 값이 복구됨
- persistence를 끄면 순수 메모리 저장소로 동작하며, 이 경우 Redis 서버를 내리면 값이 사라짐

### 프로세스 구분 주의점

- `Ctrl+C`로 내리는 대상은 **파이썬 API 서버**이지 **Redis 서버**가 아님
- 둘은 별개 프로세스임 → 파이썬 서버를 껐다 켜도 Redis 서버가 돌고 있으면 값은 그대로 남아있음
- 값의 유지/초기화는 **Redis 서버**를 내렸을 때 + persistence 설정 여부로 결정됨

## 5. Redis 용도 정리

- 캐시(cache): 느린 DB 조회를 대신해 자주 쓰는 데이터를 빠르게 제공함
- 세션 저장: 로그인 상태 등 임시 보관함
- 메시지 브로커 / 큐: 작업 분배 (예: Airflow)
- 실시간 순위표, 카운터
- Pub/Sub: 실시간 알림·채팅
- 분산 락: 여러 서버가 동시에 같은 자원을 건드리지 못하게 잠금
- → 캐시가 가장 대표적인 용도일 뿐, 존재 이유 전부는 아님

## 6. 실습 코드 (프레임워크 없는 단일 스크립트 REST API)

### 구성 요소 역할

- **API 서버 만들기** → 파이썬 표준 라이브러리 `http.server` (프레임워크 아님)
- **Redis 접근** → `redis` 패키지(redis-py). Redis는 HTTP가 아닌 자체 프로토콜(TCP)로 통신함
- **`requests`** → 만든 API를 호출/테스트하는 **클라이언트** 라이브러리임. 서버를 만드는 도구가 아님

### 서버 코드 (`app.py`)

```python
# 프레임워크 없이, 표준 라이브러리 http.server + redis-py 만으로 구성한 단일 스크립트 REST API

import json
from http.server import BaseHTTPRequestHandler, HTTPServer
import redis

# Redis 연결 (기본 localhost:6379). decode_responses=True 로 bytes 대신 문자열로 받음
r = redis.Redis(host="localhost", port=6379, decode_responses=True)


class Handler(BaseHTTPRequestHandler):
    def _json(self, code, payload):
        body = json.dumps(payload, ensure_ascii=False).encode("utf-8")
        self.send_response(code)
        self.send_header("Content-Type", "application/json; charset=utf-8")
        self.end_headers()
        self.wfile.write(body)

    # GET /kv/<key>  -> Redis 에서 값 조회
    def do_GET(self):
        parts = self.path.strip("/").split("/")
        if len(parts) == 2 and parts[0] == "kv":
            key = parts[1]
            value = r.get(key)
            if value is None:
                self._json(404, {"error": "not found", "key": key})
            else:
                self._json(200, {"key": key, "value": value})
        else:
            self._json(404, {"error": "unknown path", "path": self.path})

    # POST /kv/<key>  body: {"value": "..."}  -> Redis 에 값 저장
    def do_POST(self):
        parts = self.path.strip("/").split("/")
        if len(parts) == 2 and parts[0] == "kv":
            key = parts[1]
            length = int(self.headers.get("Content-Length", 0))
            raw = self.rfile.read(length) if length else b"{}"
            data = json.loads(raw)
            r.set(key, data.get("value"))
            self._json(200, {"ok": True, "key": key, "value": data.get("value")})
        else:
            self._json(404, {"error": "unknown path", "path": self.path})

    # 로그 조용히
    def log_message(self, *args):
        pass


if __name__ == "__main__":
    print("listening on http://localhost:8000")
    HTTPServer(("", 8000), Handler).serve_forever()
```

### Redis를 다루는 핵심 코드

```python
r = redis.Redis(host="localhost", port=6379, decode_responses=True)
r.set("user:1001", "현주")   # 저장
r.get("user:1001")           # 조회 -> "현주"
```

- `decode_responses=True`: Redis가 기본적으로 값을 bytes(`b'현주'`)로 반환하는데, 이를 문자열로 자동 변환하는 옵션임

## 7. 요청 시나리오

### 1단계 — 서버 실행 시점 (`python3 app.py`)

- `redis.Redis(...)`: Redis에 연결할 통로를 준비함
- `HTTPServer(...).serve_forever()`: 8000번 포트를 열고 요청이 올 때까지 무한 대기함

### 2단계 — 값 저장 요청 (`POST /kv/user:1001`, body `{"value": "현주"}`)

1. `http.server`가 요청을 받아 메서드가 POST인 것을 보고 `do_POST` 호출함
2. 주소 `/kv/user:1001`을 파싱해 `key = "user:1001"` 추출함
3. 요청 본문을 읽어 `json.loads`로 파싱 → `{"value": "현주"}` 획득함
4. `r.set("user:1001", "현주")` 실행 → Redis로 SET 명령이 전달되어 메모리에 저장됨
5. `{"ok": True, ...}` 를 200 상태로 응답함

### 3단계 — 값 조회 요청 (`GET /kv/user:1001`)

1. `http.server`가 GET인 것을 보고 `do_GET` 호출함
2. 주소에서 `key = "user:1001"` 추출함
3. `r.get("user:1001")` 실행 → Redis로 GET 명령을 보내 저장된 값을 받아옴
4. 값이 있으면 `{"key":..., "value":"현주"}` 를 200으로, 없으면 `{"error":"not found"}` 를 404로 응답함

### 요청 흐름 다이어그램

```mermaid
sequenceDiagram
    participant C as 클라이언트 (requests)
    participant A as API 서버 (http.server)
    participant R as Redis (메모리)

    C->>A: HTTP 요청 (POST/GET /kv/&lt;key&gt;)
    A->>R: SET / GET 명령 (redis-py)
    R-->>A: 값 반환
    A-->>C: JSON 응답
```

- API 서버는 데이터를 직접 저장하지 않음
- 들어온 HTTP 요청을 Redis 명령으로 번역해 전달하고, 결과를 받아 JSON으로 포장해 반환하는 **중개자** 역할임
- 실제 값은 전부 Redis 메모리에 존재함

## 8. 로컬 실행 방법

1. Redis 서버 설치·실행
   - mac: `brew install redis` 후 `redis-server`
   - linux: `apt install redis-server` 후 `redis-server`
2. 파이썬 패키지 설치: `pip install redis requests`
3. 서버 실행: `python3 app.py`
4. 호출: 다른 터미널에서 `requests`로 요청하거나, 브라우저로 `http://localhost:8000/kv/<key>` 접속함
